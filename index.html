01 — Foundations
Q1

What is a Large Language Model?

Trained to guess the next token, again and again.
A neural network trained on huge amounts of text to predict the next token. By doing that well repeatedly across trillions of tokens, it implicitly learns grammar, facts, and reasoning patterns — none of which were directly labeled.
next_token_demo.py
import torch
from transformers import AutoTokenizer, AutoModelForCausalLM

tok = AutoTokenizer.from_pretrained("gpt2")
model = AutoModelForCausalLM.from_pretrained("gpt2")

prompt = "The sky is"
ids = tok(prompt, return_tensors="pt").input_ids

with torch.no_grad():
logits = model(ids).logits[0, -1] # last-position logits
probs = torch.softmax(logits, dim=-1)
top = torch.topk(probs, 3)

for p, i in zip(top.values, top.indices):
print(tok.decode(i), round(p.item()*100, 1), "%")
# blue 71.2 % / a 9.4 % / the 4.1 %
ICPA · TIER 3
The next-token loop is exactly what runs when ICPA escalates to Ollama llama3.2 for the ~12% of German insurance emails that fast-path and RAG can't confidently resolve — this is the most expensive tier, which is why the whole pipeline is built to avoid reaching it.
Q2

What is a token?

Words are split into reusable subword pieces.
The unit an LLM actually reads — usually a subword piece, not a whole word. Text is split into tokens via BPE/SentencePiece, then mapped to integer IDs before anything touches the model.
tokenize.py
import tiktoken
enc = tiktoken.encoding_for_model("gpt-4")

ids = enc.encode("unbelievable")
print(ids) # [359, 6517, 712]
print([enc.decode([i]) for i in ids]) # ['un', 'believ', 'able']

# cost / context-budget check before calling an LLM API
def est_cost(text: str, usd_per_1k=0.03) -> float:
return len(enc.encode(text)) / 1000 * usd_per_1k
ICPA · ALL TIERS
Every German-language claims email gets tokenized before it ever touches TF-IDF or embeddings. Token-count budgeting matters because Presidio/spaCy PII anonymization runs pre-tokenization to keep raw PII out of anything that crosses the GDPR data-residency boundary.
Q3

What is an embedding?

Meaning as a point in space.
A vector (list of numbers) that captures meaning. Similar things sit close together — 'cat' lands near 'dog', far from 'car' — measured with cosine similarity or dot product.
embed.py
from sentence_transformers import SentenceTransformer
import numpy as np

model = SentenceTransformer("sentence-transformers/all-MiniLM-L6-v2")
vecs = model.encode(["cat", "dog", "car"])

def cosine(a, b):
return np.dot(a, b) / (np.linalg.norm(a) * np.linalg.norm(b))

print(cosine(vecs[0], vecs[1])) # cat·dog ~0.62
print(cosine(vecs[0], vecs[2])) # cat·car ~0.18
ICPA · TIER 2
Cohere multilingual embeddings turn each German claims email into a vector before it's matched against ChromaDB. Multilingual was non-negotiable — German insurance terminology needed embeddings trained across languages, not English-only.
Q4

What is the attention mechanism?

Thicker link = more attention. Context, captured.
It lets every token look at every other token and weigh how relevant each one is. This is how the model resolves context and coreference — what "sat" relates to, what "it" refers to.
attention.py
import torch, math

def scaled_dot_product_attention(Q, K, V, mask=None):
d_k = Q.size(-1)
scores = Q @ K.transpose(-2, -1) / math.sqrt(d_k)
if mask is not None:
scores = scores.masked_fill(mask == 0, float("-inf"))
weights = torch.softmax(scores, dim=-1)
return weights @ V, weights

# Q,K,V: [batch, seq_len, d_k]
out, attn_map = scaled_dot_product_attention(Q, K, V)
ICPA · TIER 2
Self-attention inside the embedding model is what makes "die Versicherung" and "der Vertrag" resolve to the same claim-type cluster in ChromaDB — context-aware similarity beats keyword matching for German compound nouns.
Q5

What is a Transformer?

A stack of N identical blocks, repeated.
The architecture behind modern LLMs: a stack of identical blocks, each pairing self-attention with a small feed-forward network, repeated N times (commonly 32–96+ layers).
transformer_block.py
import torch.nn as nn

class TransformerBlock(nn.Module):
def __init__(self, d_model=768, n_heads=12, d_ff=3072):
super().__init__()
self.attn = nn.MultiheadAttention(d_model, n_heads, batch_first=True)
self.ff = nn.Sequential(
nn.Linear(d_model, d_ff), nn.GELU(), nn.Linear(d_ff, d_model)
)
self.ln1, self.ln2 = nn.LayerNorm(d_model), nn.LayerNorm(d_model)

def forward(self, x, mask=None):
a, _ = self.attn(x, x, x, attn_mask=mask)
x = self.ln1(x + a) # residual + norm
x = self.ln2(x + self.ff(x)) # residual + norm
return x
ICPA · TIER 2/3
Both the Cohere embedding model and llama3.2 are transformer stacks under the hood — the architecture is shared, only the training objective and head differ between "embed for retrieval" and "generate a reply."
Q6

What is positional encoding / RoPE?

RoPE rotates each token by its position — the model reads the GAP between words.
Attention is order-blind by default, so position must be injected. RoPE rotates each token's Q/K vectors by an angle proportional to position, so what the model actually reads is the relative distance between tokens, not their absolute index.
rope.py
import torch

def rotary_embed(x, positions, base=10000):
# x: [seq_len, d_model] (d_model even)
d = x.shape[-1]
inv_freq = 1.0 / (base ** (torch.arange(0, d, 2).float() / d))
angles = positions[:, None] * inv_freq[None, :] # [seq, d/2]
cos, sin = angles.cos(), angles.sin()

x1, x2 = x[..., ::2], x[..., 1::2]
rotated = torch.stack([x1*cos - x2*sin, x1*sin + x2*cos], dim=-1)
return rotated.flatten(-2)
GO/GRPC SERVICE
Not directly used, but the same intuition (relative offset > absolute index) shows up in the telecom service's sliding-window sequence dedup logic for signaling messages, where what matters is the gap between message timestamps, not their raw clock value.
Q7

Encoder vs Decoder vs Encoder-Decoder?

Encoder-decoder (T5) does both: read, then write.
Encoders read the whole input at once (BERT) — great for understanding/classification. Decoders generate left-to-right (GPT) — great for generation. Encoder-decoders do both (T5) — read fully, then generate.
encoder_vs_decoder.py
from transformers import AutoModel, AutoModelForCausalLM, AutoModelForSeq2SeqLM

encoder_only = AutoModel.from_pretrained("bert-base-uncased") # classification, embeddings
decoder_only = AutoModelForCausalLM.from_pretrained("gpt2") # generation
enc_dec = AutoModelForSeq2SeqLM.from_pretrained("t5-small") # translation, summarization
ICPA · TIER 1/2
ICPA leans encoder-only for everything that isn't final generation — Cohere's embedding model and the spaCy NER pipeline are both encoder-style "understand, don't generate." Only Tier 3 needs a decoder, which is exactly why it's the expensive fallback.
02 — Inference
Q8

What is the context window?

Max tokens the model can see at once (prompt + reply). Older tokens fall outside and are forgotten.
The maximum number of tokens a model can consider at once — prompt plus reply combined. Anything beyond it gets dropped, which is why retrieval and summarization exist as workarounds.
budget_check.py
def fits_in_window(history_tokens: int, new_msg_tokens: int,
max_window=8192, reserve_for_reply=512) -> bool:
used = history_tokens + new_msg_tokens
return used + reserve_for_reply <= max_window

# sliding-window truncation when history grows too large
def truncate_history(messages: list, max_tokens: int, token_counter):
kept, total = [], 0
for m in reversed(messages): # keep most recent first
t = token_counter(m["content"])
if total + t > max_tokens: break
kept.append(m); total += t
return list(reversed(kept))
ICPA · TIER 3
Context budget is why Tier 2's retrieval matters so much — only the top-k reranked chunks from ChromaDB get passed into llama3.2's prompt, never the full claim history, to leave headroom for the response and keep latency predictable.
Q9

What is temperature?

Temperature = the randomness knob.
A knob on randomness applied during sampling. Low temperature (0–0.3) gives focused, repeatable answers; high temperature (0.8–1.2) gives more diverse and creative — and riskier — ones.
temperature.py
import torch

def sample_with_temperature(logits, temperature=1.0):
if temperature == 0:
return torch.argmax(logits) # greedy, deterministic
scaled = logits / temperature
probs = torch.softmax(scaled, dim=-1)
return torch.multinomial(probs, num_samples=1)

# API call: deterministic for extraction, creative for drafting
extraction_resp = client.messages.create(model="claude-sonnet-4-6",
temperature=0.0, max_tokens=500, messages=[...])
draft_resp = client.messages.create(model="claude-sonnet-4-6",
temperature=0.8, max_tokens=500, messages=[...])
ICPA · TIER 3
llama3.2 runs at low, near-deterministic temperature for claims replies — insurance correspondence needs consistency across runs, not creative variation. A reproducible wrong answer is debuggable; a random one isn't.
Q10

Top-k vs Top-p (nucleus) sampling?

Top-k keeps the k best · Top-p keeps the top by probability mass.
Both trim the next-token candidate pool before sampling. Top-k keeps the k most likely tokens flat; top-p keeps the smallest set whose cumulative probability adds up to p — adapting to how confident the distribution is.
topk_topp.py
import torch

def top_k_filter(logits, k=50):
v, _ = torch.topk(logits, k)
threshold = v[..., -1, None]
return torch.where(logits < threshold, float("-inf"), logits)

def top_p_filter(logits, p=0.9):
sorted_logits, sorted_idx = torch.sort(logits, descending=True)
cum_probs = torch.cumsum(torch.softmax(sorted_logits, dim=-1), dim=-1)
remove = cum_probs > p
remove[..., 1:] = remove[..., :-1].clone() # keep first token over p
remove[..., 0] = False
sorted_logits[remove] = float("-inf")
return sorted_logits.scatter(-1, sorted_idx, sorted_logits)
ICPA · TIER 3
Top-p is set conservatively for claims replies — a narrow nucleus keeps the LLM from drifting into plausible-but-wrong phrasing, which directly feeds the RAGAS faithfulness metric the team tracks per release.
03 — RAG & Retrieval
Q11

What is RAG?

Fetch real documents, then answer from them — not memory.
Retrieval-Augmented Generation: fetch relevant documents and feed them into the prompt, so the model answers from real, current, grounded data instead of relying on what it memorized at training time.
rag_minimal.py
def rag_answer(query: str, collection, llm_client, k=5) -> str:
docs = collection.query(query_texts=[query], n_results=k)
context = "\n---\n".join(docs["documents"][0])

prompt = f"""Answer using ONLY the context below.
If the answer isn't there, say you don't know.

Context:
{context}

Question: {query}"""

resp = llm_client.messages.create(
model="claude-sonnet-4-6", max_tokens=600, temperature=0.1,
messages=[{"role": "user", "content": prompt}],
)
return resp.content[0].text
ICPA · TIER 2
This is ICPA's core production tier: a three-stage LangGraph pipeline — TF-IDF fast path first, then RAG over ChromaDB with a Redis semantic cache in front of it, only falling to llama3.2 generation when retrieval confidence is too low. Getting RAGAS faithfulness from 72% → 91% came from tightening this exact loop.
Q12

What is a vector database?

A store of embeddings that finds the most similar items to a query.
A store of embeddings that finds the most similar items to a query using nearest-neighbour search (typically approximate — HNSW). It's the retrieval engine that powers RAG.

Q13

Prompting vs RAG vs Fine-tuning?

Try them top to bottom — cheapest and easiest first.
Prompting changes instructions. RAG adds external knowledge. Fine-tuning changes the model's weights. Reach for them in that order — cost and complexity rise sharply at each step.
decision_helper.py
def choose_approach(needs_fresh_data: bool, needs_new_behavior: bool,
prompt_alone_works: bool) -> str:
if prompt_alone_works:
return "prompting" # cheapest, instant iteration
if needs_fresh_data:
return "RAG" # knowledge changes often / is large
if needs_new_behavior:
return "fine-tuning" # tone, format, domain-specific reasoning
return "prompting"
ICPA · TIER 1-3
ICPA's three tiers map almost exactly to this ladder: Tier 1 (TF-IDF) is rule-based, no LLM at all; Tier 2 (RAG) handles ~95% of resolved traffic at zero external LLM cost; Tier 3 (prompted llama3.2) is the last resort. Full fine-tuning was never needed — prompting + RAG closed the gap.

What is fine-tuning?

Keep training on your data so it adapts to your task.
Continuing to train a pretrained model on a smaller, focused dataset so it specialises in a task, tone, or domain — the weights themselves change.

Q15

What is LoRA / PEFT?

LoRA / PEFT: freeze the giant, train a small add-on. Cheap.
Parameter-efficient fine-tuning: freeze the big model and train tiny low-rank adapter weights instead. Far cheaper, with almost the same quality as full fine-tuning.

Q16

What is RLHF?

Humans rank answers → model learns to match human preference.
Reinforcement Learning from Human Feedback: humans rank outputs, a reward model learns their preference, and the LLM is tuned (typically via PPO or DPO) to match that preference.

Q17

What is hallucination — and how do you reduce it?

Hallucination = fluent fiction. Ground it in real data.
When a model states something false with full confidence. Reduce it with RAG/grounding, citations, lower temperature, and explicitly allowing "I don't know" as a valid answer.

)
print(results)

Zero-shot: just ask. Few-shot: give examples first to set the pattern. Chain-of-thought: ask it to reason step by step before producing the final answer — improves accuracy on multi-step problems.

Q19

Interview Cheat Sheet — say it exactly like this

Five answers worth memorizing verbatim — these are the ones interviewers expect crisp, rehearsed delivery on, not improvisation.
Q: What is a Transformer?
"A seq-to-seq model using self-attention to process all tokens in parallel, capturing long-range dependencies without recurrence."
Q: Explain Q, K, V.
"Q is what I'm looking for, K is what I advertise, V is what I share.
Attention = softmax(QKᵀ/√d) · V"
Q: Why Multi-Head Attention?
"Runs H attention heads in parallel — each learns different relationships
(syntax, coreference, semantics). Outputs concatenated + projected."
Q: Why not RNNs?
"RNNs process tokens sequentially — slow & forget long context.
Transformers parallelize fully on GPU and capture full context via attention."
Q: What does LayerNorm + Residuals do?
"Residuals prevent vanishing gradients (output = F(x)+x).
LayerNorm stabilizes activations — enables very deep networks (96+ layers)."
DELIVERY NOTE
Memorize the shape of these answers, not the exact words — interviewers notice canned delivery. Use them as a structure (definition → mechanism → why it matters), then swap in your own phrasing live.
Q20

What is quantization?

Lower precision = much smaller model, tiny accuracy cost.
Storing model weights in lower precision (e.g. 16-bit → 4-bit). The model becomes much smaller and faster, with a small, usually acceptable, accuracy cost.
quantize.py
from transformers import BitsAndBytesConfig, AutoModelForCausalLM

bnb_config = BitsAndBytesConfig(
load_in_4bit=True, bnb_4bit_quant_type="nf4",
bnb_4bit_compute_dtype="bfloat16",
)
model = AutoModelForCausalLM.from_pretrained(
"meta-llama/Llama-3.2-3B", quantization_config=bnb_config, device_map="auto",
)
# ~4x smaller footprint, runs comfortably on consumer GPU / CPU-offload
